# Skill Manager - 使用示例

## 基本用法

### 1. 查看技能状态
```
用户: "列出所有技能"
助手: 运行技能管理器，显示所有技能列表

用户: "显示已启用的技能"
助手: 列出当前启用的技能

用户: "显示禁用的技能"
助手: 列出所有禁用的技能
```

### 2. 搜索技能
```
用户: "搜索代码审查相关的技能"
助手: 搜索并显示 code-review, pr-review-toolkit 等技能

用户: "查找前端设计技能"
助手: 找到并显示 frontend-design 技能

用户: "有哪些AI相关的技能?"
助手: 搜索AI、自动化、自我改进等技能
```

### 3. 启用/禁用技能
```
用户: "启用 article-extractor 技能"
助手: 启用该技能并确认

用户: "禁用不用的技能"
助手: 询问要禁用哪些技能，或建议不常用的技能

用户: "切换 skill-name 技能状态"
助手: 如果启用则禁用，如果禁用则启用
```

### 4. 技能信息
```
用户: "查看 code-review 技能的详细信息"
助手: 显示技能描述、大小、最后修改时间等

用户: "skill-name 技能是做什么的?"
助手: 显示技能的功能和用途
```

## 高级用法

### 1. 技能分析
```
用户: "分析技能使用情况"
助手: 显示使用统计、最常用技能、建议优化

用户: "哪些技能很少使用?"
助手: 列出使用频率低的技能，建议禁用

用户: "技能质量如何?"
助手: 分析技能文档完整性、代码质量等
```

### 2. 技能组织
```
用户: "创建开发技能包"
助手: 将开发相关技能打包，方便一键启用

用户: "切换到文档模式"
助手: 启用文档相关技能，禁用其他技能

用户: "整理技能分类"
助手: 按类别重新组织技能
```

### 3. 批量操作
```
用户: "启用所有前端相关技能"
助手: 搜索前端相关技能并全部启用

用户: "禁用所有测试技能"
助手: 找到测试相关技能并禁用

用户: "重置到默认配置"
助手: 只启用核心开发技能，禁用其他
```

## 实际场景示例

### 场景1: 开始编码工作
```
用户: "我要开始写代码了，设置一下技能"
助手: 
1. 启用开发包: code-review, commit-commands, typescript-lsp 等
2. 禁用设计包和文档包
3. 确认当前启用的技能适合开发工作
```

### 场景2: 设计UI界面
```
用户: "需要设计一个登录页面"
助手:
1. 启用 frontend-design, canvas-design, playground
2. 可选启用颜色、图标相关技能
3. 禁用不必要的开发技能
```

### 场景3: 写技术文档
```
用户: "要写项目文档"
助手:
1. 启用 article-extractor, doc-coauthoring, docx 等
2. 禁用代码开发技能
3. 设置文档写作环境
```

### 场景4: 技能清理和维护
```
用户: "感觉技能太多了，清理一下"
助手:
1. 分析技能使用频率
2. 建议禁用30天未使用的技能
3. 合并功能重复的技能
4. 更新技能索引文档
```

## 与现有脚本的集成

### 通过skill-manager调用现有脚本
```
用户: "运行技能分析报告"
助手: 调用 ./scripts/skill-analyzer.sh full

用户: "创建自定义技能包"
助手: 调用 ./scripts/skill-packs.sh create <包名> <技能列表>

用户: "更新技能索引"
助手: 调用更新脚本，刷新 SKILLS-INDEX.md
```

### 自动化工作流
```
# 早晨启动开发环境
1. 启用开发技能包
2. 检查技能更新
3. 设置开发相关配置

# 切换工作模式
1. 保存当前技能状态
2. 启用新工作模式技能包
3. 确认切换完成

# 结束工作
1. 分析当天技能使用
2. 生成使用报告
3. 建议优化配置
```

## 故障排除

### 常见问题
```
问题: "技能启用后没效果"
解决: 需要开始新的OpenClaw会话才能加载新技能

问题: "找不到某个技能"
解决: 检查技能名称是否正确，或搜索相关关键词

问题: "技能冲突了"
解决: 禁用功能重复的技能，或设置技能优先级

问题: "技能太多导致性能问题"
解决: 禁用不常用的技能，使用技能包按需加载
```

### 最佳实践
1. **按需启用** - 不要一次性启用所有技能
2. **使用技能包** - 为常见工作流创建预定义包
3. **定期清理** - 每月检查并清理不用的技能
4. **备份配置** - 备份重要的技能包配置
5. **测试新技能** - 在新会话中测试新启用的技能

## 与OpenClaw的集成

### 自动触发
skill-manager skill 会在以下情况自动触发:
- 用户提到"技能"、"管理技能"、"启用/禁用"
- 用户询问可用技能列表
- 用户需要搜索特定功能技能

### 状态持久化
- 技能启用/禁用状态保存在文件系统中
- 技能包配置保存在 skill-packs/ 目录
- 使用记录保存在 active/.skill-analysis/

### 跨会话生效
- 技能状态变更在**新会话**中生效
- 当前会话继续使用已加载的技能
- 建议重要变更后开始新会话
